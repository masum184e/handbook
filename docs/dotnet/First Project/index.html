First Project

Install .NET​

Download and install the .NET SDK from:

https://dotnet.microsoft.com/download

After installation, verify:

dotnet --version

If it prints a version number — you’re ready

Create Your First Project​

Open terminal / command prompt:

dotnet new console -n MyFirstApp

What this does:

  • dotnet new → create new project
  • console → template type
  • -n MyFirstApp → project name

Then enter the folder:

cd MyFirstApp

Run the project:

dotnet run

Output:

Hello, World!

You just ran your first .NET app!

Project Folder Structure Explained​

Inside MyFirstApp, you'll see something like:

MyFirstApp/
│
├── Program.cs
├── MyFirstApp.csproj
├── bin/
└── obj/

1. Program.cs​

This is the main entry point of your application.

In modern .NET (6+), it looks like this:

Console.WriteLine("Hello, World!");

Behind the scenes, .NET automatically wraps this in a Main method.

How it worked before .NET 6:​

using System;

namespace MyFirstApp
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello, World!");
        }
    }
}

But now it’s simplified.

2. MyFirstApp.csproj​

This is the project file.

Open it and you’ll see something like:

<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net8.0</TargetFramework>
  </PropertyGroup>

</Project>

What this means:​

  • Sdk → using Microsoft .NET SDK
  • OutputType → Executable
  • TargetFramework → .NET version (like net8.0)

This file:

  • Manages dependencies
  • Defines framework version
  • Controls build settings

Think of it as the project configuration file.

3. bin/ Folder​

Contains:

  • Compiled output
  • Executable (.exe on Windows)
  • DLL files

Created after running:

dotnet build

You normally don’t edit this manually.

4. obj/ Folder​

Temporary build files used internally by .NET.

Also auto-generated. Don’t edit this either.

Program Structure in .NET​

Now let’s expand your app.

Edit Program.cs:

Console.WriteLine("Enter your name:");
string name = Console.ReadLine();

Console.WriteLine($"Hello, {name}!");

Run:

dotnet run

You now have:

  • Input
  • Variables
  • String interpolation
  • Console output

Adding More Files (Real Structure)​

In real projects, you create separate files.

Example:

Create Person.cs​

public class Person
{
    public string Name { get; set; }

    public void Greet()
    {
        Console.WriteLine($"Hi, I'm {Name}");
    }
}

Modify Program.cs​

Person person = new Person();
person.Name = "John";
person.Greet();

Now your project structure looks like:

MyFirstApp/
│
├── Program.cs
├── Person.cs
├── MyFirstApp.csproj
├── bin/
└── obj/

This is how real projects grow.

How .NET Builds and Runs​

When you run:

dotnet run

It:

  1. Reads .csproj
  2. Compiles C# code
  3. Creates IL (Intermediate Language)
  4. Executes using .NET runtime

What Comes Next?​

After console apps, you’ll typically move to:

  • Web apps using ASP.NET Core
  • Desktop apps (WPF / WinForms)
  • Web APIs
  • Blazor apps
  • Microservices

Typical Real-World Project Structure (Bigger App)​

Example structure:

MyApp/
│
├── Controllers/
├── Models/
├── Services/
├── Data/
├── Program.cs
├── appsettings.json
└── MyApp.csproj

Used especially in web apps.
