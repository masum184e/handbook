In **C#**, data structures are used to organize and store data efficiently. Here are some of the **most commonly used data structures** you’ll see in everyday C# development.

## Array

**Array** stores elements of the **same type** in a fixed-size sequential collection.

**Key points**

- Fixed length
- Fast access by index
- Stored in contiguous memory

```csharp
int[] numbers = { 10, 20, 30, 40 };
Console.WriteLine(numbers[0]); // 10
```

**Use cases**

- Storing a known number of items
- Fast index-based access

## List

`List<T>` is a **dynamic array** that can grow or shrink.

**Key points**

- Part of `System.Collections.Generic`
- Automatically resizes
- Easy insertion and removal

```csharp
List<string> names = new List<string>();
names.Add("Alice");
names.Add("Bob");
```

**Use cases**

- When the number of elements can change
- Most commonly used collection in C#

## Dictionary

`Dictionary<TKey, TValue>` stores **key-value pairs**.

**Key points**

- Fast lookup by key
- Keys must be unique
- Internally implemented using a hash table

```csharp
Dictionary<int, string> students = new Dictionary<int, string>();
students.Add(1, "John");
students.Add(2, "Sara");

Console.WriteLine(students[1]);
```
- `containsKey()` check wheather the key exist or not.

**Use cases**

- Fast searching
- Mapping relationships (ID → Name)

## Stack

`Stack<T>` follows **LIFO (Last In, First Out)**.

**Key points**

- Push → add item
- Pop → remove last item
- Peek → see top item

```csharp
Stack<int> stack = new Stack<int>();
stack.Push(1);
stack.Push(2);
stack.Pop(); // removes 2
```

**Use cases**

- Undo/redo operations
- Expression evaluation
- Function calls

## Queue

`Queue<T>` follows **FIFO (First In, First Out)**.

**Key points**

- Enqueue → add item
- Dequeue → remove first item

```csharp
Queue<string> queue = new Queue<string>();
queue.Enqueue("Task1");
queue.Enqueue("Task2");

queue.Dequeue(); // removes Task1
```

**Use cases**

- Task scheduling
- Request processing
- Messaging systems

## LinkedList

`LinkedList<T>` stores elements as **nodes connected by pointers**.

**Key points**

- Dynamic size
- Fast insertion/deletion
- No index-based access

```csharp
LinkedList<int> list = new LinkedList<int>();
list.AddLast(10);
list.AddLast(20);
```

**Use cases**

- Frequent insertions/deletions
- Implementing queues or stacks

## Quick summary

| Data Structure | Best For               |
| -------------- | ---------------------- |
| Array          | Fixed-size data        |
| List           | Dynamic collections    |
| Dictionary     | Fast key-value lookup  |
| Stack          | LIFO operations        |
| Queue          | FIFO operations        |
| LinkedList     | Frequent insert/delete |
