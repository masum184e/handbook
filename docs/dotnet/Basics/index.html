Basics

  • int.Parse: Converts a string into an integer.

    string numberText = "25";
    int number = int.Parse(numberText);

  • string interpolation ($""): Allows inserting variables directly inside a string using {}.

    string name = "John";
    int age = 30;
    string message = $"Name: {name}, Age: {age}";

  • file path (@""): Verbatim string literal. Used to avoid escaping backslashes in file paths.

    string path = @"C:\Users\John\Documents\file.txt";
    • it also allow multiline text
    string str = @"hello first Line
    hello second line
    hello third line"
    • you can write @ and $ together also
    string userName = "Rahim";
    string path = $@"C:\Users\{userName}\Documents";

  • var: Implicitly typed variable. The compiler determines the type automatically.

    var number = 10;        // int
    var text = "Hello";     // string
    var price = 9.99;      // double
    • you must need to assign value when you declare variable with var.

namespace​

In C#, a namespace is a container used to organize code and prevent naming conflicts. It groups related classes, interfaces, structs, enums, and other namespaces into a logical hierarchy.

Think of a namespace like folders in a file system. Just as folders help organize files, namespaces help organize types in a program.

It is a logical grouping of types.

It helps to:

  • Organize large codebases
  • Avoid class name conflicts
  • Improve readability
  • Provide hierarchical structure
namespace MyApplication
{
    class Program
    {
        static void Main()
        {
            Console.WriteLine("Hello World");
        }
    }
}
  • MyApplication is the namespace
  • Program is a class inside the namespace

Why Namespaces Are Needed

Imagine two libraries both contain a class called Logger.

Without namespaces ❌:

class Logger { }
class Logger { } // Error

With namespaces ✅:

namespace ProjectA
{
    class Logger { }
}

namespace ProjectB
{
    class Logger { }
}

Now both classes can exist without conflict.

Namespace Hierarchy​

Namespaces can be nested.

namespace Company
{
    namespace Project
    {
        class MyClass
        {
        }
    }
}

This creates:

Company
   └── Project
        └── MyClass

Short form

namespace Company.Project
{
    class MyClass
    {
    }
}

Both are identical.

Using Namespaces (using keyword)​

To access classes in another namespace, we use using.

using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("Hello");
    }
}
  • System is a namespace in .NET
  • Console class is inside System

Without using:

System.Console.WriteLine("Hello");

ref Parameter​

ref allows a method to modify the value of a variable passed from the caller.

The variable must be initialized before passing it.

using System;

class Program
{
    static void AddFive(ref int num)
    {
        num += 5;
    }

    static void Main()
    {
        int x = 10;
        AddFive(ref x);
        Console.WriteLine(x); // Output: 15
    }
}
  • x is passed by reference.
  • The method directly changes the original variable.

out Parameter​

out is also used to pass variables by reference, but it is mainly used to return multiple values from a method.

  • The variable does NOT need initialization before passing.
  • The method must assign a value before returning.
static void GetValues(out int a, out int b)
{
    a = 10;
    b = 20;
}

static void Main()
{
    int x, y;
    GetValues(out x, out y);

    Console.WriteLine(x); // 10
    Console.WriteLine(y); // 20
}

Common example: TryParse

int number;
bool result = int.TryParse("123", out number);

params Parameter​

params allows a method to accept a variable number of arguments.

  • Must be the last parameter in the method.
static int Sum(params int[] numbers)
{
    int total = 0;

    foreach(int n in numbers)
    {
        total += n;
    }

    return total;
}

static void Main()
{
    Console.WriteLine(Sum(1,2,3));
    Console.WriteLine(Sum(10,20,30,40));
}
  • You can pass any number of values.

Optional Parameters​

Optional parameters allow you to omit arguments when calling a method by giving default values.

static void Greet(string name = "Guest")
{
    Console.WriteLine("Hello " + name);
}

static void Main()
{
    Greet();        // Hello Guest
    Greet("Rahim"); // Hello Rahim
}
  • If no argument is provided, the default value is used.
  • It should be placed at very end.
