- For working with money use `decimal` type. Don't forget the `m` suffix for `decimal` number, for `float` use `f` suffix.
- `float` uses less memory than `double`, but `double` is more accurate, `decimal` is most accurate

  ```csharp
  double a = 0.1 + 0.2;    // 0.30000000000000004
  decimal a = 0.1m + 0.2m; // 0.3
  ```

  - This happens because `0.1` and `0.2` cannot be represented exactly in binary. `decimal` stores numbers in base-10 (decimal) format, which avoids these rounding problems.

## Type Casting

### Implicit Casting

Small type to large type cast automatically(think like small box easily fit in large box).

```
byte → short → int → long → float → double
         ↓
       char
```

Implicit casting applied from left to right automatically.

### Don't overflow when casting

```c#
int bigValue = 300;
byte smallValue = (byte)bigValue;

Console.WriteLine(smallValue);  // 44 😱
```

Range of `byte` is 0-255, so it keep 300-256=44, which is dangerous.

When overflow occured, it stored value by comparing form begining(`0` for `byte`).

`checked` keyword check the casted value

```c#
byte result = checked((byte)(b + 1));
```

In the same way, underflow can also occur.

- `int.Parse()`: convert from string to number. It crashes when string couldn't be converted into number.
- `int.TryParse()`: safe way to convert, prevent program crash.
- `Convert`: convert by rounding number.
- ToString(): convert any data to string.
