Intensity Transformation Function​

An intensity transformation function is a mathematical operation used in digital image processing to modify the pixel intensity values of an image. It transforms each input pixel value rrr into a new output value sss using a function:

s=T(r)s = T(r)s=T(r)

where:

  • rrr = input intensity (gray level)
  • sss = output intensity after transformation
  • TTT = transformation function

These transformations are mainly used for:

  • image enhancement
  • contrast adjustment
  • brightness correction
  • highlighting details
  • image segmentation and analysis

Basic Concept​

In grayscale images:

  • Pixel values usually range from 0 to 255

    • 0 → black
    • 255 → white

The transformation function changes every pixel independently.

For example:

Input Pixel (r)Output Pixel (s)

200220

General Intensity Transformation Model​

The relationship is:

s=T(r)s=T(r)s=T(r)

Different choices of T(r)T(r)T(r) produce different enhancement effects.

Piecewise Linear Transformation​

A piecewise linear transformation is an intensity transformation in digital image processing where the mapping function is composed of multiple straight-line segments.

Instead of using one single equation for all intensity values, different equations are used over different intensity ranges.

The general idea is still:

s=T(r)s=T(r)s=T(r)
  • But now T(r)T(r)T(r) changes form in different regions.
  • Uses multiple linear segments for enhancement.

Basic Idea

A normal linear transformation uses one equation for all inputs.

y=2x+1y = 2x + 1y=2x+1

But a piecewise linear transformation uses different linear equations for different intervals.

f(x)={x+2x<02x+1x≥0f(x)= \begin{cases} x+2 & x < 0 \\ 2x+1 & x \ge 0 \end{cases}f(x)={x+22x+1​x<0x≥0​

So:

  • for negative values → use one line
  • for nonnegative values → use another line

Meaning of “Piecewise”

the function is built from several pieces.

Each piece behaves linearly inside its own region.

Contrast Stretching​

Contrast Stretching (also called Normalization) is an image enhancement technique used to improve the contrast of an image by expanding the range of intensity values.

Sometimes an image looks:

  • too dark
  • too bright
  • low contrast

because pixel intensities occupy only a small range instead of the full grayscale range.

Contrast stretching spreads the intensity values over the full available range.

Basic Idea

Suppose grayscale values should ideally range from:

0 to 2550 \text{ to } 2550 to 255

But the image actually uses only:

50 to 18050 \text{ to } 18050 to 180

Contrast stretching maps:

  • 50 → 0
  • 180 → 255

and stretches all intermediate values proportionally.

Formula

If:

  • input intensity = rrr
  • output intensity = sss
  • minimum input intensity = rminr_{min}rmin​
  • maximum input intensity = rmaxr_{max}rmax​

then:

Where:

  • LLL = number of intensity levels
  • for 8-bit image:
L=256L=256L=256 s=(r−rmin)(rmax−rmin)(L−1)s=\frac{(r-r_{min})}{(r_{max}-r_{min})}(L-1)s=(rmax​−rmin​)(r−rmin​)​(L−1)

Example

Suppose:

  • rmin=50r_{min}=50rmin​=50
  • rmax=180r_{max}=180rmax​=180

For pixel:

r=100r=100r=100

Before stretching:

  • image appears dull
  • low detail visibility

After stretching:

  • brighter details become clearer
  • dark regions become distinguishable
  • overall image quality improves

Applications

  • X-ray image enhancement
  • satellite image analysis
  • low-light photography
  • fingerprint recognition

Bit Plane Slicing​

Bit Plane Slicing separates an image into its individual binary bit planes.

Each pixel in an 8-bit image consists of 8 bits:

b7b6b5b4b3b2b1b0b_7b_6b_5b_4b_3b_2b_1b_0b7​b6​b5​b4​b3​b2​b1​b0​

Each bit position forms one bit plane.

Basic Idea

In an 8-bit grayscale image:

  • MSB (Most Significant Bit) carries major visual information
  • LSB (Least Significant Bit) carries fine details/noise

By separating bit planes:

  • important information can be analyzed
  • noise can be reduced
  • compression becomes easier

Suppose a pixel intensity is:

157157157

Binary form:

157=10011101157 = 10011101157=10011101

So:

Important Observation

Higher-order bits:

b7,b6,b5b_7,b_6,b_5b7​,b6​,b5​

contain most image structure.

Lower-order bits:

b1,b0b_1,b_0b1​,b0​

contain:

  • small details
  • noise

Visualization

An image can be decomposed into:

  • Bit Plane 7 → strongest visual shape
  • Bit Plane 6 → important structure
  • …
  • Bit Plane 0 → tiny details/noise

Applications

  • Image Compression: Store only important bit planes.
  • Watermarking: Hide information in lower bit planes.
  • Noise Reduction: Ignore noisy lower bits.
  • Pattern Recognition: Analyze significant image features.

Histogram Processing​

Histogram processing is a fundamental technique in digital image processing used to analyze and improve image quality by manipulating the distribution of pixel intensities.

It is mainly used for:

  • contrast enhancement
  • brightness adjustment
  • image normalization
  • feature extraction
  • image segmentation

What is an Image Histogram​

An image histogram is a graphical representation of the distribution of pixel intensity values in an image.

For a grayscale image:

  • x-axis → gray levels (0–255)
  • y-axis → number of pixels at each gray level

Mathematical Representation​

If:

  • rkr_krk​ = intensity level
  • nkn_knk​ = number of pixels having intensity rkr_krk​
  • nnn = total number of pixels

then the histogram is:

where:

  • p(rk)p(r_k)p(rk​) = probability of occurrence of gray level rkr_krk​

Understanding Histogram Shapes​

Dark Image​

Histogram concentrated toward the left side.

  • most pixels are dark
  • low intensity values dominate

Bright Image​

Histogram concentrated toward the right side.

  • high intensity values dominate

Low Contrast Image​

Histogram occupies a narrow range.

  • image looks dull or faded

High Contrast Image​

Histogram spread across the full range.

  • image appears clear and sharp

Histogram Processing Techniques​

Main techniques include:

  1. Histogram Equalization
  2. Histogram Matching (Specification)
  3. Local Histogram Processing

Histogram Equalization​

Histogram equalization is a contrast enhancement technique that redistributes image intensity values so that they span the full available range more uniformly.

It is widely used in image processing to improve visibility in low-contrast images.

Basic Idea​

Suppose:

  • the image has poor contrast
  • most pixel intensities are concentrated within a narrow range

Histogram equalization transforms the image so that intensity values are spread over the entire range.

As a result:

  • dark regions become more distinguishable
  • bright regions gain detail
  • overall contrast improves

Important Note

In discrete digital images, histogram equalization does not usually produce a perfectly uniform histogram because:

  • the number of pixels is finite
  • gray levels are discrete
  • rounding introduces quantization effects

Histogram equalization therefore produces an approximately equalized histogram rather than an ideal continuous uniform distribution.

Histogram Matching/Specification​

Histogram matching transforms an image so that its histogram matches a specified histogram.

Unlike equalization, here the target histogram is predefined.

Process​

Steps:

  1. Equalize the input image
  2. Equalize the desired histogram
  3. Map between the two

Example​

Suppose we want:

  • dark image transformed into brighter image

We choose a target histogram concentrated toward high intensities.

Result:

  • image brightness increases
  • specific visual appearance achieved

Applications​

  • medical imaging
  • satellite imagery
  • cinematic color grading

Local Histogram Processing​

Global histogram methods use the entire image.

Local histogram processing operates on small neighborhoods.

Why Needed?​

Sometimes:

  • different regions have different illumination
  • global enhancement fails

Local processing enhances each region independently.

Method​

For every pixel:

  1. choose neighborhood window
  2. compute local histogram
  3. apply enhancement

Example​

In a shadowed image:

  • dark regions become clearer
  • bright regions remain balanced

Applications​

  • face recognition
  • fingerprint enhancement
  • surveillance systems

Histogram Statistics​

Histogram statistics are numerical descriptors extracted from an image histogram that summarize the distribution of pixel intensities. In image processing and pattern recognition, histogram statistics transform raw image data into compact features used for enhancement, segmentation, classification, texture analysis, and object recognition.

Think of a histogram as a probability distribution of image gray levels. Histogram statistics describe its shape: where values concentrate, how spread out they are, whether they are symmetric, and whether they contain multiple peaks.

Image Histogram Fundamentals​

For a grayscale image with intensity levels 0 to L-1:

  • nkn_knk​: number of pixels with intensity rkr_krk​
  • N: total number of pixels

The histogram:

h(rk)=nkh(r_k)=n_kh(rk​)=nk​

Normalized histogram:

p(rk)=nkNp(r_k)=\frac{n_k}{N}p(rk​)=Nnk​​

This behaves like a probability distribution:

∑k=0L−1pr(rk)=1\sum_{k=0}^{L-1}p_r(r_k)=1k=0∑L−1​pr​(rk​)=1

Histogram statistics are computed from pr(rk)p_r(r_k)pr​(rk​).

Why histogram statistics matter​

Two images can have identical dimensions but very different intensity distributions.

Examples:

  • Dark image → most pixels near 0
  • Bright image → most pixels near 255
  • High contrast → intensities spread widely
  • Low contrast → intensities clustered

Histogram statistics quantify these properties. Pattern recognition systems use them as features.

Applications:

  • Face recognition
  • Medical imaging
  • Texture classification
  • Defect detection
  • Remote sensing
  • Image retrieval

N-th Central Moment​

The nth moment of rrr about its mean is called the nth central moment in statistics and image processing.

μn=E[(r−m)n]\mu_n = E[(r - m)^n]μn​=E[(r−m)n]

For a discrete histogram:

μn=∑k=0L−1(rk−m)np(rk)\mu_n = \sum_{k=0}^{L-1} (r_k - m)^n p(r_k)μn​=k=0∑L−1​(rk​−m)np(rk​)

Where:

  • rkr_krk​ = intensity level
  • p(rk)p(r_k)p(rk​) = probability (normalized histogram)
  • μ\muμ = mean intensity
  • nnn = order of the moment

This measures how pixel intensities are distributed relative to the mean brightness.

Different values of nnn describe different properties:

First central moment n=1n = 1n=1

μ1=E[r−m]=0\mu_1 = E[r - m] = 0μ1​=E[r−m]=0
  • Always zero
  • Because deviations above and below the mean cancel out

Second central moment n=2n = 2n=2

μ2=E[(r−m)2]\mu_2 = E[(r - m)^2]μ2​=E[(r−m)2]
  • This is the variance
  • Measures contrast / spread

Third central moment n=3n = 3n=3

μ3=E[(r−m)3]\mu_3 = E[(r - m)^3]μ3​=E[(r−m)3]
  • Measures skewness
  • Indicates asymmetry:
    • Positive → darker side dominates
    • Negative → brighter side dominates

Fourth central moment n=4n = 4n=4

  • Related to kurtosis
  • Measures “peakedness” or “flatness” of histogram
