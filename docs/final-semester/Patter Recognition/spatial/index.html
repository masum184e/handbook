Spatial Filtering​

Spatial filtering is a technique in digital image processing where image pixels are modified based on the values of neighboring pixels.

It is widely used for:

  • image enhancement
  • noise reduction
  • edge detection
  • sharpening
  • blurring
  • feature extraction

The operation is performed directly in the spatial domain.

Spatial Domain

In the spatial domain, image processing operations are applied directly to image pixels.

An image is represented as:

f(x,y)f(x,y)f(x,y)

where:

  • x,yx,yx,y = pixel coordinates
  • f(x,y)f(x,y)f(x,y) = intensity at that pixel

Spatial filtering modifies pixel values using nearby pixels.

Rule of Padding​

For a kernel of size m×nm \times nm×n:

  • horizontal padding: n−12\frac{n-1}{2}2n−1​​
  • vertical padding: m−12\frac{m-1}{2}2m−1​​

So total padding added is:

  • left/right → n−12\frac{n-1}{2}2n−1​​
  • top/bottom → m−12\frac{m-1}{2}2m−1​​

The center of the kernel is aligned with each image pixel, multiplying overlapping entries and summing them to produce the output pixel.

Kernel​

A kernel is a small matrix used in image processing, computer vision, and pattern recognition to perform operations such as:

  • Smoothing
  • Sharpening
  • Edge detection
  • Texture extraction
  • Feature enhancement
  • Pattern recognition

It is also called:

  • Filter
  • Mask
  • Window
  • Convolution matrix

Kernels are fundamental in both classical image processing and modern deep learning systems such as CNNs.

Conditions for Kernel Separability

A kernel is separable if:

  • Matrix rank = 1
  • Rows/columns are linearly dependent

Charecterstics of Separable Kernels

Separable filtering requires 2D structure

  • Horizontal pass smooths left–right
  • Vertical pass smooths up–down
  • Together they form full 2D Gaussian smoothing

Think of it like this:

  • First pass → averages along rows
  • Second pass → averages along columns
  • Result → full neighborhood averaging

Separable filtering works because:

2D kernel=(1D column)×(1D row)2D\ kernel = (1D\ column)\times(1D\ row)2D kernel=(1D column)×(1D row)

So instead of doing one expensive 2D2D2D operation, we do:

  • one horizontal pass
  • one vertical pass

and still get the same result.

Correlation vs Convolution​

Spatial filtering may use:

  • Correlation: Mask applied directly.
  • Convolution: Mask rotated 180∘180^\circ180∘ before application.

Most image processing uses convolution.

Correlation​

Correlation measures the similarity between:

  • an image region
  • and a kernel/template

The kernel is applied without flipping.

Mathematical form:

g(x,y)=∑s=−aa∑t=−bbw(s,t)f(x+s,y+t)g(x,y)=\sum_{s=-a}^{a}\sum_{t=-b}^{b} w(s,t)f(x+s,y+t)g(x,y)=s=−a∑a​t=−b∑b​w(s,t)f(x+s,y+t)

Where:

  • f(x,y)f(x,y)f(x,y) = input image
  • w(s,t)w(s,t)w(s,t) = kernel
  • g(x,y)g(x,y)g(x,y) = output image

Convolution​

Convolution is similar to correlation, but the kernel is first rotated by 180∘180^\circ180∘ before sliding over the image.

Mathematical form:

g(x,y)=∑s=−aa∑t=−bbw(s,t)f(x−s,y−t)g(x,y)=\sum_{s=-a}^{a}\sum_{t=-b}^{b} w(s,t)f(x-s,y-t)g(x,y)=s=−a∑a​t=−b∑b​w(s,t)f(x−s,y−t)

Notice the negative signs. This represents kernel flipping.

Smoothing (Low Pass Filter)​

Smoothing (also called blurring or low-pass filtering) is a fundamental operation in image processing and pattern recognition used to reduce noise and small variations in data while preserving the overall structure of the image.

In simple terms, smoothing makes an image “less sharp” by reducing abrupt intensity changes between neighboring pixels.

Why smoothing is used

Real-world images often contain:

  • Sensor noise (random pixel intensity fluctuations)
  • Compression artifacts
  • Fine texture that is not important for analysis

Smoothing helps:

  • Improve object detection and segmentation
  • Reduce false edges in edge detection
  • Stabilize feature extraction in pattern recognition
  • Preprocess data for machine learning models

Non-Linear Spatial Filtering​

Non-linear filters do not compute output as weighted sums. Instead they apply operations like:

  • median
  • maximum
  • minimum
  • ranking
  • logical decisions

They violate linearity conditions.

General form:

g(x,y)=T[N(x,y)]g(x,y)=T[N(x,y)]g(x,y)=T[N(x,y)]

where TTT is some non-linear transformation on neighborhood NNN.

These filters are especially useful when image corruption is non-Gaussian.

Robert filter​

The Robert filter (more correctly called the Roberts Cross operator) is one of the earliest and simplest edge detection techniques used in Pattern Recognition and Image Processing. It is used to find edges in images by approximating the gradient (rate of intensity change) using very small convolution kernels.

The Roberts Cross operator detects edges by computing the approximate gradient of image intensity using diagonal differences.

It highlights regions where pixel intensity changes sharply → these are typically edges.

It works on 2×2 neighborhoods, making it extremely fast but also sensitive to noise.

Kernels

The Roberts filter uses two 2×2 convolution kernels:

  1. Gx kernel (diagonal 1)
Gx=[100−1]G_x = \begin{bmatrix} 1 & 0 \\ 0 & -1 \end{bmatrix}Gx​=[10​0−1​]

Gy kernel (diagonal 2)

Gy=[01−10]G_y = \begin{bmatrix} 0 & 1 \\ -1 & 0 \end{bmatrix}Gy​=[0−1​10​]

How it works

For each 2×2 block in the image:

  1. Apply Gx → measures diagonal difference (top-left to bottom-right)
  2. Apply Gy → measures diagonal difference (top-right to bottom-left)
  3. Combine results:
G=Gx2+Gy2G = \sqrt{G_x^2 + G_y^2}G=Gx2​+Gy2​​

or sometimes simplified as:

G=∣Gx∣+∣Gy∣G = |G_x| + |G_y|G=∣Gx​∣+∣Gy​∣

  1. Compute Gy

  1. Gradient magnitude

Using Euclidean form:

G=(−50)2+02=50G = \sqrt{(-50)^2 + 0^2} = 50G=(−50)2+02​=50

So the edge strength at that location is 50.

  • High value (like 50, 100, etc.) → strong edge
  • Low value (near 0) → smooth region (no edge)

The Roberts operator detects diagonal edges, because it compares diagonal pixel pairs.

Why it works

Edges are places where intensity changes rapidly. Roberts filter captures this by:

  • subtracting diagonal neighbors
  • approximating gradient in 45° rotated axes

So it is essentially a discrete derivative operator.

Unsharp Masking​

Unsharp masking is a classic image enhancement technique used in image processing and pattern recognition to increase perceived sharpness by emphasizing edges and fine details.

Despite the name, it does the opposite of what it sounds like: it uses a blurred (unsharp) version of the image to enhance sharpness.

Core Idea

An image contains:

  • Low-frequency components → smooth regions (background, lighting variations)
  • High-frequency components → edges, textures, fine details

Unsharp masking works by:

  1. Creating a blurred version of the image (removing high frequencies)
  2. Subtracting it from the original → extracting edges (high-frequency details)
  3. Adding those details back to the original image → sharpening

So it enhances:

“What the image lost when it was blurred”

In pattern recognition systems, unsharp masking is used as preprocessing to improve feature quality.

  • improves segmentation and recognition accuracy
