## Introduction

### Image Processing

**Goal:** Improve or modify an image.

- The computer **changes the image itself** (enhance, filter, compress, etc.).
- It usually **does not try to understand what the image contains**.

**Examples:**

- Removing noise from a photo
- Increasing brightness/contrast
- Blurring or sharpening an image
- Compressing images (like JPEG)

**Simple idea:**
_Input image → modify image → output image_

Example: A phone camera automatically improving photo quality.

### Computer Vision

**Goal:** Make computers **understand and interpret images**.

- The computer tries to **extract meaning** from images or videos.
- Often uses **machine learning and AI**.

**Examples:**

- Face recognition
- Self-driving cars detecting pedestrians
- Medical image diagnosis
- Object detection in security cameras

**Simple idea:**
_Input image → analyze → output information_

Example: Detecting whether a picture contains a **cat or dog**.

### Computer Graphics

**Goal:** **Create images** using computers.

- Instead of analyzing images, it **generates them**.
- Used heavily in **games, animation, and 3D modeling**.

**Examples:**

- 3D characters in video games
- Animated movies
- Virtual reality environments
- Rendering 3D models

**Simple idea:**
_Input data (models, shapes) → generate image_

Example: Rendering a 3D scene in a video game.

### Quick Comparison

| Field                 | Main Goal             | Input       | Output             |
| --------------------- | --------------------- | ----------- | ------------------ |
| **Image Processing**  | Modify/enhance images | Image       | Improved image     |
| **Computer Vision**   | Understand images     | Image/video | Information/labels |
| **Computer Graphics** | Create images         | Models/data | Image/animation    |

**Easy way to remember:**

- **Computer Graphics** → _Make images_
- **Image Processing** → _Improve images_
- **Computer Vision** → _Understand images_


## Types of Image

In **image processing**, “types of images” are typically classified based on how pixel information is represented and stored. Here’s a structured breakdown of the main categories:

> Memorize max common 3-4 items

### 1. Binary Images (Bi-level Images)

* **Definition**: Each pixel has only two possible values: 0 or 1.
* **Representation**: Black and white (no gray levels).
* **Bit depth**: 1 bit per pixel.
* **Use cases**:
  * Document scanning (text vs background)
  * Shape extraction
* **Example**: Printed text converted into pure black-and-white pixels.

### 2. Grayscale Images

* **Definition**: Pixels represent intensity (brightness) only.
* **Range**: Typically 0–255
  * 0 = black
  * 255 = white
* **Bit depth**: Usually 8 bits per pixel.
* **Use cases**:
  * Medical imaging (X-rays)
  * Edge detection and feature extraction
* **Key idea**: No color information, only luminance.

### 3. Color Images (RGB Images)

* **Definition**: Pixels contain color information using channels.
* **Common model**: RGB (Red, Green, Blue)
* **Representation**:
  * Each pixel = (R, G, B)
  * Each channel usually 8 bits → total 24 bits per pixel
* **Use cases**:
  * Photography
  * Computer vision applications
* **Example**: Digital photos captured by cameras.

### 4. Indexed Images

* **Definition**: Pixels store an index to a color table (palette), not actual color values.
* **Components**:
  * Index matrix
  * Color lookup table (LUT)
* **Advantage**: Reduces memory usage.
* **Use cases**:
  * GIF images
  * Simple graphics

### 5. Multispectral Images

* **Definition**: Images captured at multiple wavelengths beyond visible light.
* **Bands**: More than 3 channels (e.g., infrared, ultraviolet).
* **Use cases**:
  * Satellite imaging
  * Agriculture monitoring
* **Example**: Remote sensing data.

### 6. Hyperspectral Images

* **Definition**: Advanced form of multispectral with **hundreds of narrow spectral bands**.
* **Feature**: Detailed spectral signature for each pixel.
* **Use cases**:
  * Mineral detection
  * Environmental monitoring
* **Key difference**: Much higher spectral resolution than multispectral.

### 7. Thermal Images

* **Definition**: Represent heat (infrared radiation) emitted by objects.
* **Pixel values**: Temperature-related intensity.
* **Use cases**:
  * Night vision
  * Industrial inspection
* **Example**: Detecting heat leaks in buildings.

### 8. Depth Images (Range Images)

* **Definition**: Each pixel represents distance from the sensor.
* **Units**: Usually in meters or millimeters.
* **Use cases**:
  * 3D reconstruction
  * Robotics (e.g., obstacle detection)
* **Example**: Data from depth cameras like LiDAR.

### 9. Volumetric Images (3D Images)

* **Definition**: Stack of 2D images forming a 3D dataset.
* **Representation**: Voxels instead of pixels.
* **Use cases**:
  * CT scans
  * MRI imaging

### 10. HDR (High Dynamic Range) Images

* **Definition**: Capture a wider range of brightness levels than standard images.
* **Feature**: Preserves detail in both dark and bright regions.
* **Use cases**:
  * Professional photography
  * Visual effects

## Digital vs Analog Image
In image processing, the distinction between **analog** and **digital images** is fundamental because it determines how images are captured, stored, and processed.

### Analog Image

An **analog image** is a **continuous representation** of a scene, where intensity values vary smoothly over space.

#### Key Characteristics

* **Continuous in space and intensity**
* No discrete pixels
* Infinite resolution (theoretically)
* Represented by physical phenomena (light, voltage, film density)

#### Examples

* Photographic film
* X-ray films
* Signals from analog cameras (before digitization)

#### Mathematical View

An analog image can be modeled as a continuous function:
$$
f(x, y)
$$
where:
* (x, y) → spatial coordinates (continuous)
* (f) → intensity (continuous)

### Digital Image

A **digital image** is a **discrete representation** of an image, made up of pixels with finite intensity values.

#### Key Characteristics

* **Discrete in space and intensity**
* Stored as a matrix of numbers
* Finite resolution
* Suitable for computer processing

#### Representation

A digital image is:
$$
f(x, y) \rightarrow f[i, j]
$$
where:

* (i, j) → pixel indices
* Each pixel has a quantized value (e.g., 0–255)

### Key Differences

| Feature        | Analog Image      | Digital Image         |
| -------------- | ----------------- | --------------------- |
| Nature         | Continuous        | Discrete              |
| Representation | Function (f(x,y)) | Matrix (f[i,j])       |
| Storage        | Physical medium   | Digital memory        |
| Processing     | Difficult         | Easy (computer-based) |
| Noise          | More sensitive    | Can be filtered       |

## Digital Image Processing

Digital Image Processing refers to the use of computational techniques to perform operations on a **digital image (pixel-based representation)** in order to:

* Enhance visual appearance
* Analyze and interpret image content

A digital image is treated as a **matrix of pixel values**, and DIP applies mathematical operations to that matrix.


**Applications of Digital Image Processing**

* Medical imaging (X-rays, MRI)
* Remote sensing (satellite images)
* Security & surveillance (face detection)
* Autonomous vehicles
* Industrial inspection
* Multimedia (photos, videos)

**Simple Example**

When you use a smartphone camera:

1. Image is captured digitally
2. DIP enhances brightness and colors
3. Noise is reduced automatically
4. Face detection may be applied

> Using algorithms to process and analyze digital images for enhancement and information extraction.


**Goals of Digital Image Processing (DIP)**

1. Image Enhancement (Improve Visual Quality)
2. Image Restoration (Correct Degradations)
3. Image Compression (Reduce Data Size)
4. Image Segmentation (Partitioning)
5. Feature Extraction
6. Image Recognition and Interpretation
7. Image Reconstruction
8. Visualization
9. Measurement and Analysis

### Types of Operations in DIP

#### Low-Level Processing

* Input: image → Output: image
* Example: smoothing, sharpening

#### Mid-Level Processing

* Input: image → Output: features
* Example: segmentation, edge detection

#### High-Level Processing

* Input: features → Output: understanding
* Example: object recognition

## Sources of Image

In **Digital Image Processing**, images originate from different **acquisition sources** depending on how the energy (light, radiation, or signals) is captured and converted into a visual form. Understanding these sources is important because each has distinct characteristics (resolution, noise, spectral range).

> Memorize max common 3-4 items

### 1. Visible Light Imaging (Optical Sources)

* Uses **visible spectrum (≈ 400–700 nm)**.
* Captured by standard cameras using lenses and sensors.

**Examples**: Digital cameras, Smartphone cameras, CCTV systems

**Characteristics**

* Produces RGB (color) images
* High spatial resolution
* Affected by lighting conditions

### 2. Infrared (IR) Imaging

* Captures radiation in the **infrared spectrum** (beyond visible light).
* Types: Near-infrared, Thermal infrared

**Examples**: Night vision systems, Heat detection cameras

**Characteristics**

* Works in low/no light
* Shows temperature variations

### 3. X-ray Imaging

* Uses **X-ray radiation** to penetrate objects and capture internal structures.

**Examples**: Medical X-rays, Airport security scanners

**Characteristics**

* High penetration power
* Produces grayscale images
* Useful for internal inspection

### 4. Ultrasound Imaging

* Uses **high-frequency sound waves**.
* Echoes are used to construct images.

**Examples**: Medical sonography (pregnancy scans)

**Characteristics**

* Real-time imaging
* Safe (non-ionizing)
* Lower resolution compared to X-rays

### 5. Magnetic Resonance Imaging (MRI)

* Based on nuclear magnetic resonance principles.

**Examples**: Brain and organ imaging in hospitals

**Characteristics**

* High contrast for soft tissues
* No ionizing radiation
* Expensive and complex

### 6. Radar Imaging

* Uses **radio waves** to detect objects and form images.

**Examples**: Weather radar, Air traffic control, Military surveillance

**Characteristics**

* Works in all weather conditions
* Can penetrate clouds and rain

### 7. Satellite Imaging (Remote Sensing)

* Captured from satellites orbiting Earth.
* **Types**: Multispectral, Hyperspectral

**Examples**: Weather monitoring, Earth observation

**Characteristics**

* Covers large areas
* Includes non-visible bands (IR, UV)

### 8. Microscopic Imaging

* Uses microscopes to capture very small objects.

**Examples**: Biological cells, Materials science imaging

**Characteristics**

* Very high magnification
* Can be optical or electron-based

### 9. Electron Imaging

* Uses electron beams instead of light.

**Examples**; Scanning Electron Microscope (SEM), Transmission Electron Microscope (TEM)

**Characteristics**

* Extremely high resolution
* Used for atomic-level imaging

### 10. Computer-Generated Images

* Not captured from physical sources; created using algorithms.

**Examples**: Graphics in games, Simulations, 3D rendering

**Characteristics***

* Fully synthetic
* Controlled environment

## Fundamental steps in DIP

It form a structured pipeline that transforms a raw image into meaningful information. These steps are typically sequential, but in practice they may overlap depending on the application.

![Fundamental steps in DIP](/img/image_processing/fundamental_steps_in_dip.png)

### 1. Image Acquisition

* The first step: capturing and digitizing the image.
* Involves sensors (camera, scanner) and conversion from analog to digital.

**Key Processes**

* Sampling (spatial discretization)
* Quantization (intensity discretization)

**Output**

* A digital image (pixel matrix)

### 2. Image Enhancement

* Improves visual appearance for human viewing.

**Techniques**

* Contrast stretching
* Histogram equalization
* Noise filtering (mean, median filters)
* Sharpening (edge enhancement)

**Goal**

* Highlight important features and improve clarity

### 3. Image Restoration

* Recovers the original image from degraded input using mathematical models.

**Common Problems**

* Blur (motion blur, defocus)
* Noise (Gaussian, salt-and-pepper)

**Techniques**

* Deconvolution
* Wiener filtering

**Key Difference**

* Enhancement = subjective
* Restoration = objective (model-based)

### 4. Color Image Processing

* Deals with processing of color images.

**Tasks**

* Color correction
* Color space conversion (RGB → HSV, etc.)
* False color rendering

### 5. Wavelets and Multiresolution Processing

* Represents images at multiple scales/resolutions.

**Uses**

* Image compression
* Feature extraction
* Denoising

**Example**

* Wavelet transform

### 6. Image Compression

* Reduces data size for storage/transmission.

**Types**: Lossless (PNG, TIFF), Lossy (JPEG)

**Techniques**

* Transform coding (DCT)
* Run-length encoding

### 7. Morphological Processing

* Processes images based on shapes (mainly binary images).

**Operations**

* Dilation
* Erosion
* Opening and closing

**Application**

* Object extraction and boundary detection

### 8. Image Segmentation

* Divides image into meaningful regions.

**Methods**

* Thresholding
* Edge-based segmentation
* Region-based segmentation

**Goal**

* Simplify image for analysis

### 9. Representation and Description (Feature Extraction)

* Converts segmented regions into useful features.

**Features**

* Shape (area, perimeter)
* Texture
* Color

**Purpose**

* Prepare data for recognition

### 10. Object Recognition / Interpretation

* Assigns meaning to objects in the image.

**Examples**

* Face recognition
* Character recognition (OCR)

**Techniques**

* Machine learning
* Pattern recognition

### 11. Knowledge Base

* Central component that guides all processing steps.

**Role**

* Provides prior information
* Controls algorithms and parameters

**Example**

* Database of known object patterns

## Components of an Image Processing

It refer to the **hardware and software elements** required to acquire, process, store, and display images. This is a system-level view (different from processing steps).

```
Scene → Sensor → Digitizer → Computer + Software → Storage → Display / Output
```

### 1. Image Acquisition System

* Responsible for capturing the image.

**Components**

* Sensor (CCD/CMOS camera)
* Digitizer (Analog-to-Digital Converter)

**Function**

* Converts real-world scene → digital image

### 2. Specialized Image Processing Hardware

* Dedicated hardware for fast image processing.

**Examples**

* GPUs
* DSP (Digital Signal Processors)
* FPGA-based systems

**Function**

* Performs computationally intensive tasks efficiently

### 3. Computer (Host System)

* General-purpose system controlling the processing.

**Role**

* Executes algorithms
* Controls input/output devices

**Examples**

* PCs, workstations, embedded systems
### 4. Image Processing Software

* Programs and algorithms used to process images.

**Functions**

* Enhancement
* Segmentation
* Analysis

**Examples**

* MATLAB
* OpenCV (library)
* Python-based tools

### 5. Mass Storage System

* Stores large volumes of image data.

**Types**

* Short-term storage (RAM)
* Long-term storage (Hard disk, SSD)
* Archival storage (Cloud, databases)

**Function**

* Efficient storage and retrieval of images

### 6. Image Display Devices

* Devices used to visualize images.

**Examples**

* Monitors (LCD/LED)
* Projectors

**Function**

* Convert digital data → visible image

### 7. Hardcopy Devices

* Produce physical output of images.

**Examples**

* Printers
* Film recorders

**Function**

* Digital image → printed image

### 8. Networking / Communication Systems

* Enables transmission of images between systems.

**Examples**

* Internet
* Local networks

**Function**

* Share and transfer image data
