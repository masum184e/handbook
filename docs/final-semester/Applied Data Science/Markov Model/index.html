Markov Model

The Markov property is a fundamental idea that describes memoryless behavior in random systems.

The future state depends only on the current state — not on how the system got there.

Video Tutorial

Stochastic process​

A stochastic process is a sequence of random variables evolving over time:

X0,X1,X2,…X_0, X_1, X_2, \dotsX0​,X1​,X2​,…

Each (XtX_tXt​) represents the state of a system at time (ttt).

Conditional Probability​

We want to predict the future:

P(Xt+1∣Xt,Xt−1,…,X0)P(X_{t+1} \mid X_t, X_{t-1}, \dots, X_0)P(Xt+1​∣Xt​,Xt−1​,…,X0​)

This means:

Probability of next state given the entire history

In general probability theory:

  • The future could depend on everything
  • So the safest form is:
P(future∣full past)P(\text{future} \mid \text{full past})P(future∣full past)

So at this stage, nothing is simplified yet.

The key idea (memoryless property)​

The Markov property says: The future depends only on the present, not the past.

P(Xt+1∣Xt,Xt−1,…,X0)=P(Xt+1∣Xt)P(X_{t+1} \mid X_t, X_{t-1}, \dots, X_0) = P(X_{t+1} \mid X_t)P(Xt+1​∣Xt​,Xt−1​,…,X0​)=P(Xt+1​∣Xt​) P(Tomorrow∣Today, Yesterday, ...)=P(Tomorrow∣Today)P(\text{Tomorrow} \mid \text{Today, Yesterday, ...}) = P(\text{Tomorrow} \mid \text{Today})P(Tomorrow∣Today, Yesterday, ...)=P(Tomorrow∣Today)

This is the heart of the Markov property.

Meaning

  • You don’t need the full history
  • Only the current state matters for predicting the next state

Without Markov property​

You would normally expect: The more past information you have, the better your prediction

So:

P(Tomorrow∣Today, Yesterday)≠P(Tomorrow∣Today)P(\text{Tomorrow} \mid \text{Today, Yesterday}) \neq P(\text{Tomorrow} \mid \text{Today})P(Tomorrow∣Today, Yesterday)=P(Tomorrow∣Today)

Left and right side are not equal as they are not follow markov property.

With Markov property​

We assume: Past history adds no extra useful information

So:

P(Tomorrow∣Today, Yesterday, ...)=P(Tomorrow∣Today)P(\text{Tomorrow} \mid \text{Today, Yesterday, ...}) = P(\text{Tomorrow} \mid \text{Today})P(Tomorrow∣Today, Yesterday, ...)=P(Tomorrow∣Today)

Because the system is assumed to be memoryless.

That means

  • All relevant information from the past is already captured in the present state
  • The present state is a complete summary of history

Types of Markov Models​

  1. Discrete-Time Markov Chain (DTMC)
    • Time moves in steps (t = 1, 2, 3...)
    • Most common type
  2. Continuous-Time Markov Chain (CTMC)
    • Time is continuous
  3. Hidden Markov Model (HMM)
    • States are not directly observable
    • Used in speech recognition, NLP

Markov Chain​

A Markov chain is a process where:

  • Time is discrete (steps)
  • The Markov property holds

It is defined by:

  • A set of states
  • Transition probabilities between states
π1=π0⋅P\pi_1 = \pi_0 \cdot Pπ1​=π0​⋅P πt=π0⋅Pt\pi_t = \pi_0 \cdot P ^ tπt​=π0​⋅Pt
  • πt\pi_tπt​ = probability distribution at time (ttt)
  • PPP = transition matrix
  • Multiplication applies one-step transitions to evolve the system forward

Components of a Markov Model​

States​

All possible conditions the system can be in.

Example: Weather = {Sunny, Rainy}

Transition Probabilities​

Probability of moving from one state to another

Current → NextSunny (S)Rainy (R)
Sunny (S)0.80.2
Rainy (R)0.40.6

Transition Matrix

P=[0.80.20.40.6]P = \begin{bmatrix} 0.8 & 0.2 \\ 0.4 & 0.6 \end{bmatrix}P=[0.80.4​0.20.6​]

Weather Prediction​

Let’s build a simple Markov model.

Initial State​

Suppose today is Sunny:

π0=[1,0]\pi_0 = [1, 0]π0​=[1,0]

Predict Tomorrow​

π1=π0⋅P\pi_1 = \pi_0 \cdot Pπ1​=π0​⋅P π1=[1,0]⋅[0.80.20.40.6]=[0.8,0.2]\pi_1 = [1, 0] \cdot \begin{bmatrix} 0.8 & 0.2 \\ 0.4 & 0.6 \end{bmatrix} = [0.8, 0.2]π1​=[1,0]⋅[0.80.4​0.20.6​]=[0.8,0.2]

Interpretation:

  • 80% chance Sunny tomorrow
  • 20% chance Rainy tomorrow

Predict Day After Tomorrow​

π2=π1⋅P\pi_2 = \pi_1 \cdot Pπ2​=π1​⋅P π2=[0.8,0.2]⋅P=[0.72,0.28]\pi_2 = [0.8, 0.2] \cdot P = [0.72, 0.28]π2​=[0.8,0.2]⋅P=[0.72,0.28]

Interpretation:

  • 72% Sunny
  • 28% Rainy

Two-step prediction​

We use transitions step-by-step:

From Sunny → Sunny → Sunny:

0.8×0.8=0.640.8 \times 0.8 = 0.640.8×0.8=0.64

From Sunny → Rainy → Sunny:

0.2×0.4=0.080.2 \times 0.4 = 0.080.2×0.4=0.08

Total probability of Sunny after 2 days:

0.64+0.08=0.720.64 + 0.08 = 0.720.64+0.08=0.72

From Sunny → Sunny → Rainy:

0.8×0.2=0.160.8 \times 0.2 = 0.160.8×0.2=0.16

From Sunny → Rainy → Rainy:

0.2×0.6=0.120.2 \times 0.6 = 0.120.2×0.6=0.12

Total probability of Rainy after 2 days:

0.16+0.12=0.280.16 + 0.12 = 0.280.16+0.12=0.28

Key insight from example

Even for future predictions:

  • We only use the current state at each step
  • We don’t go back to the full history

Intuition

Think of it like this:

  • A goldfish memory system 🐟
  • It forgets everything except where it is now
  • That’s enough to decide the next move

Applications and Importance​

Real-Life Applications

Markov models are widely used in:

  1. Google Search & Ranking: PageRank algorithm (based on Markov chains)
  2. Speech Recognition: Hidden Markov Models (HMM)
  3. Finance: Stock price modeling
  4. Biology: DNA sequence analysis
  5. Games & AI: Decision making in uncertain environments

Importance

It simplifies complex systems because:

  • No need to track entire history
  • Easier modeling and computation
  • Widely used in AI, finance, physics, etc.

Steady-State (Long-Term Behavior)​

After many steps, probabilities stabilize.

We solve:

This gives long-term probabilities of being in each state.

Question​

1. What is the Markov Property?

The Markov property states that the future state depends only on the current state, not on the full history of past states.

P(Xt+1∣Xt,Xt−1,…,X0)=P(Xt+1∣Xt)P(X_{t+1} \mid X_t, X_{t-1}, \dots, X_0) = P(X_{t+1} \mid X_t)P(Xt+1​∣Xt​,Xt−1​,…,X0​)=P(Xt+1​∣Xt​)

Example: Weather prediction - If today is rainy, tomorrow’s weather depends only on today’s condition, not on whether it rained last week.

2. Define Transition Matrix

A transition matrix is a square matrix that contains probabilities of moving from one state to another in a Markov chain.

P=[0.70.30.40.6]P = \begin{bmatrix} 0.7 & 0.3 \\ 0.4 & 0.6 \end{bmatrix}P=[0.70.4​0.30.6​]

Key properties

  • Each row sums to 1
  • All entries are between 0 and 1
  • Rows represent current state, columns represent next state

5. Absorbing State

An absorbing state is a state that once entered cannot be left.

Mathematically:

Pii=1P_{ii} = 1Pii​=1

Example

  • Game "win" state — once you win, you stay in win state.

Significance

  • Models terminal conditions
  • Used in game theory and survival processes
6. Classification Question

Determine whether the following Markov chain is:

  • Irreducible
  • Periodic or aperiodic

Explain your reasoning.

Given:

P=[0.70.30.40.6]P = \begin{bmatrix} 0.7 & 0.3 \\ 0.4 & 0.6 \end{bmatrix}P=[0.70.4​0.30.6​]
  • Irreducible? Yes — every state can be reached from every other state.
  • Periodic or aperiodic? Aperiodic — because self-loops exist (non-zero diagonal entries).
7. Trick Question

If a system depends on both current and previous states:

P(St+1∣St,St−1)P(Sₜ₊₁ | Sₜ, Sₜ₋₁)P(St+1​∣St​,St−1​)

Can it still be modeled as a Markov chain?
If yes, how?

Yes, it can still be modeled as a Markov chain.

How?

Convert it into an expanded state:

Instead of:

P(St+1∣St,St−1)P(S_{t+1} \mid S_t, S_{t-1})P(St+1​∣St​,St−1​)

Define new state:

Yt=(St,St−1)Y_t = (S_t, S_{t-1})Yt​=(St​,St−1​)

Now:

P(Yt+1∣Yt)P(Y_{t+1} \mid Y_t)P(Yt+1​∣Yt​)

Conclusion: By expanding the state space, it becomes a first-order Markov chain.
