Hidden Markov Model

A Hidden Markov Model (HMM) is an extension of a Markov model where:

The system has hidden (unobservable) states, but we can observe outputs that depend on those states.

Video Explaination

Key Idea​

In a normal Markov model:

  • You directly see the state (e.g., weather)

In a Hidden Markov Model:

  • You cannot see the actual state
  • You only see observations generated by that state

So you infer the hidden state from observations.

Real-Life Intuition​

Imagine:

  • You cannot see the weather
  • But you observe whether people carry umbrellas

Umbrella = observation
Weather = hidden state

Components of an HMM​

An HMM is defined by 5 main elements:

1. Hidden States (S)​

Set of possible internal states Example:

S=Sunny,RainyS = {Sunny, Rainy}S=Sunny,Rainy

2. Observations (O)​

What we actually see

O=Umbrella,No UmbrellaO = {Umbrella, No\ Umbrella}O=Umbrella,No Umbrella

3. Transition Probabilities (A)​

Probability of moving between hidden states:

A=[P(S→S)P(S→R)P(R→S)P(R→R)]A = \begin{bmatrix} P(S \to S) & P(S \to R) \\ P(R \to S) & P(R \to R) \end{bmatrix}A=[P(S→S)P(R→S)​P(S→R)P(R→R)​]

4. Emission Probabilities (B)​

Probability of observing something given a state:

B=[P(Umbrella∣Sunny)P(No∣Sunny)P(Umbrella∣Rainy)P(No∣Rainy)]B = \begin{bmatrix} P(Umbrella|Sunny) & P(No|Sunny) \\ P(Umbrella|Rainy) & P(No|Rainy) \end{bmatrix}B=[P(Umbrella∣Sunny)P(Umbrella∣Rainy)​P(No∣Sunny)P(No∣Rainy)​]

5. Initial Probabilities (π)​

Starting probability of each state:

π=[P(Sunny),P(Rainy)]\pi = [P(Sunny), P(Rainy)]π=[P(Sunny),P(Rainy)]

Complete Example​

Hidden States:​

  • Sunny (S)
  • Rainy (R)

Observations:​

  • Umbrella (U)
  • No Umbrella (N)

Step 1: Define Probabilities​

Transition Matrix (A):

A=[0.70.30.40.6]A = \begin{bmatrix} 0.7 & 0.3 \\ 0.4 & 0.6 \end{bmatrix}A=[0.70.4​0.30.6​]

Emission Matrix (B):

B=[0.10.90.80.2]B = \begin{bmatrix} 0.1 & 0.9 \\ 0.8 & 0.2 \end{bmatrix}B=[0.10.8​0.90.2​]

Meaning:

  • If Sunny → 10% chance umbrella, 90% no umbrella
  • If Rainy → 80% umbrella, 20% no umbrella

Initial State:

π=[0.6,0.4]\pi = [0.6, 0.4]π=[0.6,0.4]

Problem Example​

You observe:

(Umbrella,Umbrella,No Umbrella)(Umbrella, Umbrella, No\ Umbrella)(Umbrella,Umbrella,No Umbrella)

Question: What is the most likely weather sequence?

How It Works (Conceptually)​

There are 3 main problems in HMM:

Evaluation Problem​

What is the probability of observations?

Solved using: Forward Algorithm

Decoding Problem​

What is the most likely hidden state sequence?

Solved using: Viterbi Algorithm

Learning Problem​

How to estimate model parameters?

Solved using: Baum-Welch Algorithm

Step-by-Step Intuition (Decoding)​

Observation:

Umbrella → Umbrella → No Umbrella

We compare possible hidden sequences:

Possible paths:​

  • Sunny → Sunny → Sunny
  • Sunny → Rainy → Sunny
  • Rainy → Rainy → Sunny
  • etc.

Each path has probability:

P=Initial×Transition×EmissionP = Initial \times Transition \times EmissionP=Initial×Transition×Emission

Example Path:​

Rainy → Rainy → Sunny

Probability:

=π(R)×B(R,U)×A(R,R)×B(R,U)×A(R,S)×B(S,N)= \pi(R) \times B(R,U) \times A(R,R) \times B(R,U) \times A(R,S) \times B(S,N)=π(R)×B(R,U)×A(R,R)×B(R,U)×A(R,S)×B(S,N) =0.4×0.8×0.6×0.8×0.4×0.9= 0.4 \times 0.8 \times 0.6 \times 0.8 \times 0.4 \times 0.9=0.4×0.8×0.6×0.8×0.4×0.9

This gives a relatively high probability

Why HMM is Powerful​

Because it lets you:

  • Infer hidden patterns
  • Work with noisy/incomplete data
  • Model time-dependent systems

Applications​

Speech Recognition​

  • Words = hidden states
  • Audio signal = observations

Natural Language Processing​

  • POS tagging (noun, verb, etc.)

Bioinformatics​

  • DNA/protein sequence analysis

Finance​

  • Market regimes (bull/bear)

Key Difference from Markov Chain​

FeatureMarkov ModelHidden Markov Model
States visible?Yes❌ No
ObservationsSame as statesDifferent
ComplexitySimpleMore powerful

Simple Analogy​

Think of a detective problem:

  • You can’t see the criminal (hidden state)
  • You only see clues (observations)
  • You try to infer what really happened

That’s exactly what HMM does.

Summary​

  • HMM has hidden states + observable outputs

  • Uses:

    • Transition probabilities
    • Emission probabilities

  • Solves:

    • Prediction
    • Decoding
    • Learning problems

  • Widely used in AI, speech, NLP, and biology
