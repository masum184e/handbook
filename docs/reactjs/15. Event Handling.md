React uses a system called Synthetic Events, which:

- Wraps native browser events
- Works the same across all browsers
- Has a consistent API

Key differences with HTML:

- Use camelCase (`onClick`)
- Pass a function reference, not a string
- Use `{}` to embed JavaScript

## Event Handling in Class Components

```jsx
class ClickClass extends React.Component {
  handleClick = () => {
    console.log("Clicked");
  };

  render() {
    return <button onClick={this.handleClick}>Click</button>;
  }
}
```

Arrow function avoids manual binding of `this`.

## Common Mistakes

| Mistake                            | Why it's wrong           |
| ---------------------------------- | ------------------------ |
| `onClick="handleClick"`            | React expects a function |
| `onClick={handleClick()}`          | Executes immediately     |
| Not using `event.preventDefault()` | Page reloads             |
| Uncontrolled inputs                | Harder to manage         |

Event handlers must be passed as a function reference, not executed immediately. That’s why passing arguments needs a wrapper function.

- `<button onClick={handleClick(id)}>Click</button>` runs during render, not on click.

## Using `bind()` to Pass Arguments (Less Common Today)

```jsx
function BindExample() {
  function handleClick(name) {
    alert(`Hello ${name}`);
  }

  return <button onClick={handleClick.bind(null, "Alice")}>Greet</button>;
}
```

- `bind()` returns a new function
- First argument (`null`) is `this` (important in class components)
- Works fine, but arrow functions are usually clearer

## Passing Arguments in Class Components

```jsx
class ClassExample extends React.Component {
  handleClick = (id) => {
    console.log("Clicked ID:", id);
  };

  render() {
    return <button onClick={() => this.handleClick(5)}>Click Me</button>;
  }
}
```

- Arrow function keeps `this` bound correctly
- Same concept as functional components

## Common Mistakes

- Immediate Execution: `<button onClick={handleClick(1)}>Click</button>`
- Forgetting Event Parameter: `onClick={() => handleClick(event)} // event is undefined`
- Using Inline Logic Too Much: `onClick={() => doThis() && doThat() && doMore()}`

Keep handlers readable and simple.

People often ask:

> “Does creating arrow functions in JSX hurt performance?”

- In most apps, the impact is negligible
- Clarity > micro-optimization
- Optimize only when rendering huge lists
