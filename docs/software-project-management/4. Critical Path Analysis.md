## What is Critical Path Analysis (CPA)?

**Critical Path Analysis (CPA)** is a scheduling technique used to determine:

* The **longest path** through a project network
* The **minimum project completion time**
* Which activities are **critical** (cannot be delayed without delaying the project)

CPA is widely used in construction, IT projects, manufacturing, product launches, etc.

## STEP-BY-STEP PROCESS

### STEP 1: List All Activities

Break the project into tasks.

Each activity must include:

* Activity ID
* Description
* Duration
* Immediate predecessor(s)

### STEP 2: Identify Dependencies

Define logical relationships:

| Type                 | Meaning                                      |
| -------------------- | -------------------------------------------- |
| Finish → Start (FS)  | Task B starts after A finishes (most common) |
| Start → Start (SS)   | Task B starts when A starts                  |
| Finish → Finish (FF) | Task B finishes when A finishes              |
| Start → Finish (SF)  | Rare case                                    |

*(Traditional CPA mainly uses Finish-to-Start relationships.)*

### STEP 3: Draw Network Diagram

Two main methods:

1. **Activity-on-Node (AON)** – Most common (activities in boxes)
2. **Activity-on-Arrow (AOA)** – Activities on arrows

We will use **AON** in this example.

#### COMPLETE EXAMPLE

**Project: Build a Simple Website**

| ID | Activity             | Duration (Days) | Predecessor |
| -- | -------------------- | --------------- | ----------- |
| A  | Requirement Analysis | 3               | —           |
| B  | Design               | 4               | A           |
| C  | Backend Development  | 6               | B           |
| D  | Frontend Development | 5               | B           |
| E  | Integration          | 2               | C, D        |
| F  | Testing              | 3               | E           |
| G  | Deployment           | 1               | F           |


### STEP 4: Forward Pass (Calculate Early Times)

We calculate:

* **ES (Early Start)**
* **EF (Early Finish)**

#### Formula:

```
EF = ES + Duration
ES (next activity) = Max EF of predecessors
```

#### Forward Pass Table

| Activity | Duration | ES | EF |
| -------- | -------- | -- | -- |
| A        | 3        | 0  | 3  |
| B        | 4        | 3  | 7  |
| C        | 6        | 7  | 13 |
| D        | 5        | 7  | 12 |
| E        | 2        | 13 | 15 |
| F        | 3        | 15 | 18 |
| G        | 1        | 18 | 19 |

**Project Duration = 19 days**


### STEP 5: Backward Pass (Calculate Late Times)

Now calculate:

* **LS (Late Start)**
* **LF (Late Finish)**

#### Formula:

```
LS = LF − Duration
LF (previous activity) = Min LS of successors
```

#### Backward Pass Table

| Activity | Duration | LF | LS |
| -------- | -------- | -- | -- |
| G        | 1        | 19 | 18 |
| F        | 3        | 18 | 15 |
| E        | 2        | 15 | 13 |
| C        | 6        | 13 | 7  |
| D        | 5        | 13 | 8  |
| B        | 4        | 7  | 3  |
| A        | 3        | 3  | 0  |

---

### STEP 6: Calculate Float (Slack)

#### Total Float Formula:

```
Float = LS − ES
OR
Float = LF − EF
```

#### Float Table

| Activity | ES | EF | LS | LF | Float |
| -------- | -- | -- | -- | -- | ----- |
| A        | 0  | 3  | 0  | 3  | 0     |
| B        | 3  | 7  | 3  | 7  | 0     |
| C        | 7  | 13 | 7  | 13 | 0     |
| D        | 7  | 12 | 8  | 13 | 1     |
| E        | 13 | 15 | 13 | 15 | 0     |
| F        | 15 | 18 | 15 | 18 | 0     |
| G        | 18 | 19 | 18 | 19 | 0     |

### STEP 7: Identify Critical Path

Critical Path = Activities with **Zero Float**

#### Critical Path:

**A → B → C → E → F → G**

Duration = **19 days**

## All Possible Path Combinations

From Start to Finish:

1. A → B → C → E → F → G = 19 days ✅ (Critical Path)
2. A → B → D → E → F → G = 18 days

Longest path = Critical Path

## Types of Float (All Combinations)

### 1️⃣ Total Float (TF)

Time activity can be delayed without delaying project.

### 2️⃣ Free Float (FF)

Delay allowed without affecting next activity.

Formula:

```
FF = ES(next) − EF(current)
```

### 3️⃣ Independent Float (IF)

Rarely used. Accounts for predecessor and successor flexibility.

Formula:

```
IF = ES(next) − LF(previous) − Duration
```

## 🔄 Different Network Structures (All Major Combinations)

CPA can include:

### 1. Linear Network

A → B → C → D

### 2. Merge Activities

A → C  
B → C

(Must take MAX in forward pass)

### 3. Burst Activities

A → B  
A → C

(Must take MIN in backward pass)

### 4. Parallel Paths

Two or more independent sequences

### 5. Multiple Critical Paths

If two paths have equal longest duration

Example:
Path 1 = 20 days
Path 2 = 20 days
Both are critical

## ⚙️ Special Situations in CPA

### Case 1: Lag Time

If activity has lag:

```
ES = EF(previous) + Lag
```

### Case 2: Lead Time

If overlap:

```
ES = EF(previous) − Lead
```

### Case 3: Dummy Activities (AOA only)

Used to show dependency without duration.

## 📈 PERT vs CPA

| CPA                | PERT                  |
| ------------------ | --------------------- |
| Fixed duration     | Probabilistic         |
| Deterministic      | Uses 3 time estimates |
| Construction focus | R&D focus             |

PERT formula:

```
TE = (O + 4M + P) / 6
```

## Summary Formula Sheet

Forward Pass:

```
EF = ES + Duration
ES = Max(EF of predecessors)
```

Backward Pass:

```
LS = LF − Duration
LF = Min(LS of successors)
```

Float:

```
TF = LS − ES
FF = ES(next) − EF(current)
```
